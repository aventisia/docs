# Format DateTime

The **Format DateTime** node helps you change how a date or time appears. You can choose from preset formats like "Short Date" or "Long Time," or you can define your own custom format (like `yyyy-MM-dd`). 

This is helpful when you need dates and times to match a specific format for reports, exports, or user-friendly displays.

{/* ![Format DateTime](./format-datetime.png) */}

## Configuration Options

| Field Name             | Description                                                                 | Input Type | Required? | Default Value     |
| ---------------------- | --------------------------------------------------------------------------- | ---------- | --------- | ----------------- |
| **DateTime Value**     | The date and time you want to format.                                       | Text       | No        | _(empty)_         |
| **Format Option**      | Choose whether to use a standard format or define your own custom format.   | Select     | No        | Standard          |
| **Standard Format**    | Select from common built-in formats like Short Date or Long Time.           | Select     | No        | ShortDate         |
| **Custom Format String** | Type your own pattern, like `yyyy/MM/dd HH:mm`. Only shows if using "Custom". | Text       | No        | _(empty)_         |

## Expected Output Format

The output is a **text string** showing the formatted date and/or time.

Examples:

- `"07/09/2025"` (Short Date)
- `"Wednesday, July 9, 2025"` (Long Date)
- `"2025-07-09T15:30:00"` (Sortable format)
- `"20250709"` (Custom format: `yyyyMMdd`)

The output is available as **FormattedDateTime**.

## Step-by-Step Guide

<Steps>
### Step 1

Add the **Format DateTime** node to your workflow.

### Step 2

In the **DateTime Value** field, enter the date and time you want to format. You can type it directly or use output from another node.

### Step 3

In the **Format Option** dropdown, choose one of the following:

- **Standard Format**: Select from predefined options like "Short Date" or "Long Time."
- **Custom Format**: Allows you to define your own pattern.

### Step 4

If you selected **Standard Format**, the **Standard Format** dropdown will appear. Choose a format that fits your need.

### Step 5

If you selected **Custom Format**, the **Custom Format String** field will appear. Enter your format, like `yyyy-MM-dd` or `dd/MM/yyyy hh:mm tt`.

### Step 6

The output will be available as **FormattedDateTime**, ready for use in other nodes.

</Steps>

<Callout type="info" title="Tip">
  Only one of **Standard Format** or **Custom Format String** will be shown, depending on your choice in **Format Option**.
</Callout>

## Common Mistakes & Troubleshooting

| Problem                                             | Solution                                                                                          |
| --------------------------------------------------- | ------------------------------------------------------------------------------------------------- |
| **No output shown**                                 | Make sure you entered a valid DateTime value in the **DateTime Value** field.                    |
| **Custom format not working**                       | Double-check your format string for typos (e.g., use `MM` for month, not `mm` which is minutes). |
| **Wrong format appears**                            | Verify that you chose the correct **Format Option** (Standard or Custom).                        |
| **Standard/Custom options not appearing**           | These appear based on what you choose in **Format Option**. Try switching and see them load.     |

## Real-World Use Cases

- **Reporting**: Convert system-generated dates to friendly formats like "July 9, 2025" for reports.
- **Exporting Data**: Ensure exported CSV files use consistent formats like `yyyy-MM-dd`.
- **User Interfaces**: Show only the time (e.g., `03:00 PM`) or date in a clear, readable way.
- **APIs and Integrations**: Convert date/time into ISO-like format (`2025-07-09T15:30:00`) for third-party tools.
