# Extract Text from PDF (PDFPig)

The **Extract Text from PDF (PDFPig)** node is designed to retrieve text content from PDF files efficiently. It is useful when you need to access the text data in PDFs for reviewing, editing, or leveraging the text content in other workflows or reports. This function is particularly beneficial in scenarios where PDF documents need to be transformed for content analysis or data extraction purposes.

{/* ![Extract Text from PDF (PDFPig)](./extract-text-from-pdf-pdfpig.png) */}

## Configuration Options

| Field Name         | Description                              | Input Type | Required? | Default Value |
| ------------------ | ---------------------------------------- | ---------- | --------- | ------------- |
| **SourcePdfPath**  | The path to the source PDF file.         | Text       | Yes       | _(empty)_     |
| **Extracted Text** | Returns the extracted text from the PDF. | Output     | No        | ExtractedText |

## Expected Output Format

The output of this node is **plain text** that represents the textual content extracted from the source PDF file.

## Step-by-Step Guide

<Steps>
### Step 1

Add the **Extract Text from PDF (PDFPig)** node to your workflow.

### Step 2

In the **SourcePdfPath** field, enter the complete path to the PDF file from which you want to extract text.

### Step 3

The extracted text will automatically be available as **ExtractedText**. You can use this output for further processing in your flow.

</Steps>

<Callout type="info" title="Tip">
  Ensure that the file path provided in **SourcePdfPath** is correct and
  accessible to avoid extraction errors.
</Callout>

## Common Mistakes & Troubleshooting

| Problem               | Solution                                                                                                |
| --------------------- | ------------------------------------------------------------------------------------------------------- |
| **Invalid file path** | Verify that the path in **SourcePdfPath** is correct, includes the file name, and uses forward slashes. |
| **No text extracted** | Check if the PDF is not password-protected and contains selectable text, rather than scanned images.    |
| **File not found**    | Ensure the PDF file exists at the specified path and is not moved or deleted.                           |

## Real-World Use Cases

- **Digitizing Documents**: Extract text from a scanned contract for digital archiving and searchability.
- **Generating Content Summaries**: Retrieve text from PDF reports for quick summaries or key points analysis.
- **Automated Data Collection**: Collect data from multiple PDFs to input into databases or spreadsheets for research or analytics.
