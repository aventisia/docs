# Delete FTP File

The **Delete FTP File** node is designed to remove specified files from your FTP server. This function is helpful when you need to manage files on a server and want to automate the process of clearing out unnecessary or outdated files.

For example:

- Remove temporary files after processing them.
- Clean up old backups or logs automatically.

{/* ![Delete FTP File](./delete-ftp-file.png) */}

## Configuration Options

| Field Name         | Description                       | Input Type | Required? | Default Value |
| ------------------ | --------------------------------- | ---------- | --------- | ------------- |
| **FTP Connection** | The FTP connection object.        | Text       | Yes       | _(empty)_     |
| **File Names**     | The list of file names to delete. | Text       | Yes       | _(empty)_     |

## Expected Output Format

The output of this node is a confirmation message or status indicating the success or failure of the file deletion operation.

## Step-by-Step Guide

<Steps>
### Step 1

Add **Delete FTP File** node to your workflow.

### Step 2

In the **FTP Connection** field, enter the name of the FTP connection object. Make sure this is correctly configured to connect to your FTP server.

### Step 3

In the **File Names** field, specify the files you wish to delete from the server. This can be a single file or a list of multiple files separated by commas.

### Step 4

Check that all file names are correct to avoid unintended deletions.

### Step 5

Execute the workflow to delete the specified files from your FTP server.

</Steps>

<Callout type="warning" title="Warning">
  Be cautious when specifying file names. Once a file is deleted, this action
  cannot be undone.
</Callout>

## Common Mistakes & Troubleshooting

| Problem                      | Solution                                                                          |
| ---------------------------- | --------------------------------------------------------------------------------- |
| **FTP Connection Fails**     | Ensure the FTP credentials are correctly configured and the server is reachable.  |
| **File Not Found Error**     | Double-check the file names for typos or incorrect paths.                         |
| **Unexpected File Deletion** | Verify the list of files specified for deletion before executing the node action. |

## Real-World Use Cases

- **Automated Cleanup**: Regularly remove temporary files or processed data from a shared FTP server to free up space.
- **Maintain Server Organization**: Periodically delete older versions of files after new uploads, keeping only the latest versions.
- **Security Management**: Automatically delete sensitive files from the server after a specified period to maintain data privacy.
