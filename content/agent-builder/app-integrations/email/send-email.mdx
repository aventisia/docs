# Send Email

The **Send Email** node allows you to automate email sending within your business workflows. You can utilize this feature to notify stakeholders, deliver reports, or pass along important information without manually drafting an email each time.

{/* ![Send Email](./send-email.png) */}

## Configuration Options

| Field Name                           | Description                                                                         | Input Type | Required? | Default Value |
| ------------------------------------ | ----------------------------------------------------------------------------------- | ---------- | --------- | ------------- |
| **From**                             | The sender's email address.                                                         | Text       | Yes       | _(empty)_     |
| **To**                               | The recipient's email address(es), separated by commas.                             | Text       | Yes       | _(empty)_     |
| **CC**                               | Email addresses for CC recipients, separated by commas.                             | Text       | No        | _(empty)_     |
| **BCC**                              | Email addresses for BCC recipients, separated by commas.                            | Text       | No        | _(empty)_     |
| **Subject**                          | The subject line of the email.                                                      | Text       | Yes       | _(empty)_     |
| **Body**                             | The main content of your email.                                                     | Text       | Yes       | _(empty)_     |
| **Body Format**                      | Choose the email format: Plain text or HTML.                                        | Select     | No        | Text          |
| **Attachments**                      | File paths for attachments, separated by commas.                                    | Text       | No        | _(empty)_     |
| **SMTP Server**                      | URL of the SMTP server for email sending.                                           | Text       | Yes       | _(empty)_     |
| **Server Port**                      | Port number for the SMTP server (default is 587).                                   | Text       | No        | 587           |
| **Security Protocol**                | Security protocol for the connection (e.g., SSL, TLS).                              | Select     | No        | Auto          |
| **SMTP Server Needs Authentication** | Does the SMTP server require login? Toggle this option if authentication is needed. | Switch     | No        | true          |
| **User Name**                        | Username required for SMTP authentication.                                          | Text       | No        | _(empty)_     |
| **Password**                         | Password needed for SMTP authentication.                                            | Text       | No        | _(empty)_     |
| **Accept Untrusted Certificates**    | Allow untrusted SSL certificates if necessary.                                      | Switch     | No        | false         |
| **Timeout (seconds)**                | The time period to wait for the connection to establish (default is 30 seconds).    | Text       | No        | 30            |

## Expected Output Format

The output of the **Send Email** node is a confirmation message indicating that the email was successfully dispatched, or an error message if the operation could not be completed.

## Step-by-Step Guide

<Steps>
### Step 1

Add the **Send Email** node into your automation flow.

### Step 2

Enter the sender's email address in the **From** field.

### Step 3

In the **To** field, input the recipient's email addresses. For multiple recipients, separate each address with a comma.

### Step 4

(Optional) Enter any CC and BCC email addresses in the respective fields, using commas to separate multiple addresses.

### Step 5

Fill in the **Subject** line to give your email a purpose or topic.

### Step 6

Compose the **Body** of your email with the main message content.

### Step 7

Choose the **Body Format** depending on whether you want plain text or HTML formatting.

### Step 8

(Optional) Provide paths to any files you want to attach in the **Attachments** field, separated by commas if necessary.

### Step 9

Input the **SMTP Server** URL and ensure the correct **Server Port** is specified.

### Step 10

Configure the **Security Protocol** to fit your security requirements.

### Step 11

Review if **SMTP Server Needs Authentication** is enabled, supplying **User Name** and **Password** if needed.

### Step 12

Decide whether to accept **Untrusted Certificates** based on your security protocol.

### Step 13

Set a **Timeout** if a longer connection time is required by your server (default is 30 seconds).

</Steps>

<Callout type="info" title="Tip">
  Always double-check recipient addresses and SMTP server credentials to avoid
  failed email transmissions.
</Callout>

## Common Mistakes & Troubleshooting

| Problem                   | Solution                                                                                                                                                       |
| ------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Email not sent**        | Verify SMTP configuration details, recipient addresses, and network connectivity. Ensure no typos are present in configuration inputs.                         |
| **Authentication errors** | Check that the **User Name** and **Password** are correct and that authentication is enabled when required by the SMTP server.                                 |
| **Email marked as spam**  | Ensure the email content is clear and reputable. Verify that sending limits from the SMTP server are not being exceeded, and consider adding SPF/DKIM records. |

## Real-World Use Cases

- **Automate Notification**: Instantly notify team members of system updates or alerts.
- **Streamlined Reporting**: Send daily/weekly/monthly reports to managers or clients.
- **Customer Service**: Automatically reply to customer inquiries with pre-set templates.
- **Event Invitations**: Send personalized invites for webinars, meetings, or events.
